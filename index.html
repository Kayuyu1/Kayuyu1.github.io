1、判断对象是否为垃圾的算法

引用计数法：

  • 判断对象引用数量来决定对象是否可以被回收
  • 每一个对象实例都有一个引用计数器，被引用则+1，完成引用则-1
  • 任何引用计数器为0的对象实例可以被当作垃圾回收

优点：简单，效率高

缺点：无法判断循环引用，可能会造成内存泄露。

可达分析算法：

  • 通过GC ROOT对象为根起始点，向下探索
  • 探索的路径为引用链
  • 当一个对象到达GC ROOT没有任何引用链时判断为不可达对象，为垃圾。

对象object 5、object 6、object 7虽然互相有关联，但是他们到GC Roots是不可达的，所以他们将会被判定为是可回收的对象。

优点：可以很好的解决对象相互循环引用的问题。

在Java语言中，可作为GC Roots的对象包括下面几种：

  • 虚拟机栈（栈帧的本地变量表）中引用的对象。
  • 方法区中类静态属性引用的对象。
  • 方法区中常量引用的对象。
  • 本地方法栈中JNI（即一般说的Native方法）引用的对象。

2、垃圾回收算法

2.1、标记清除算法

  • 标记：从根集合进行扫描,对存活的对象进行标记
  • 清除：回收堆中不可达对象

缺点：会造成的内存碎片。

内存碎片：不连续的内存

2.2、复制算法

  • 分为对象面和空闲面
  • 对象在对象面中
  • 存活对象从对象面复制到空闲面
  • 对象面清除所有对象
  • 对象面和空闲面相互转化

优点：解决了内存碎片问题，在对象存活率低的场景下，效率高

缺点：不得不空出50%的内存，在对象存活率高的场景下，多复制会降低效率。

2.3、标记整理

在标记清除算法的基础上，进行了回收整理。

优点：避免了内存碎片，不用分配50%空闲的空闲内存，适用于对象存活率高的场景。

缺点：多了一次整理过程，时间消耗比标记清除多。

2.4、分代收集算法

​ 分代收集算法是一套组合拳，在堆的新生区等区域用不同的算法进行回收。这样能更好的提高jvm的垃圾回收效率。

  • 新生代：Minor gc 采用了复制算法。

    新生代分为：Eden区域和两个幸存区。

  • 老年代：当对象经历一定次数的Minor gc（默认15次）后会进入老年代，或者幸存区放不下了会直接进入老年代。Major GC（Major GC通常是跟full GC是等价的，即收集整个GC堆）采用标记整理算法/标记清理算法。

2.5、什么时候触发Full GC

  • 老年代空间不足。
  • 永久代空间不足（JDK7以前）。
  • 调用System.gc(),只是提醒。回不回收是虚拟机决定。
  • 新生代对象晋升到老年代的时候，老年代剩余空间不足。

2.6、垃圾回收器前提知识

  1. Stop-the-Word：垃圾回收器在进行垃圾回收的时候，会暂停所有线程。一般的jvm优化就是指降低暂停时间。
  2. SafePoint：安全点，GC算法分析回收的时间点，设置过少就会影响回收效率，过多会增加暂停时间。一般在方法调用，循环break，异常抛出点设置。
  3. JVM的模式:
    1. Server:重量级别的jvm。对程序有更多的优化，启动慢，但稳定后，程序执行比Client快
    2. Client:轻量级别的jvm，启动快。

2.7、新生代常见的收集器

https://blog.csdn.net/weixin_43228814/article/details/88934939

  • Serial收集器（串行收集器，复制算法，-XX：+UserSerialGC）：单线程回收，简单高效，Client模式下默认的年轻代收集器。
  • ParNew收集器（复制算法，-XX：+UserParNewGC）：多线程回收，在多核执行下才有优势。
  • Parallel Scavenge收集器（并行清除收集器，复制算法，-XX：+UserParallelGC）：比起关注用户线程停顿时间，更关注系统吞吐量，在多核执行下才有优势，是Server模式下默认年轻代收集器

2.8、老年代常见收集器

  • Serial Old收集器（标记整理，-XX：+UserSerialOldGC）：特点于Serial相同，只是算法不同
  • Parallel Old收集器（标记整理，-XX：+UserParallelOldGC）：多线程，吞吐量优先
  • CMS收集器（标记清除，-XX:UserConcMarkSweepGC）：垃圾回收线程与用户执行线程同时工作，停顿时间短。缺点，采用标记清除，会造成内存碎片。

2.9、G1收集器（复制+标记整理，-XX：UseG1GC）

  • 并发和并行
  • 关注整个堆
  • 分代收集（因为G1收集器关注整个堆）
  • 空间整合
  • 可预见的停顿

2.10、GC相关面试题

2.10.1、finalize()方法

finalize()的作用是给对象提供一次逃生的机会。

  • 当对象不再被引用的时候，GC会进行第一次标记。
  • 然后调用对象的finalize()方法。
  • 如果对象在finalize()方法中再次被引用，就可避免回收。
  • 如果没有，就会被GC再次标记，回收。

不建议使用finalize()方法，因为执行它的线程优先级低，不能保证完全执行。

2.10.2、Java中强引用，软引用，弱引用，虚引用有什么用

引用类型 被垃圾回收时间 用途 生存时间
强引用 从不回收 对象一般状态 JVM停止
软引用 内存不足的时候 对象缓存 内存不足时终止
弱引用 在垃圾回收时 对象缓存 gc运行后终止
虚引用 Unknown 标记,哨兵（一般用来观察gc什么时候执行） Unknown

内存溢出和内存泄露的区别

发表于 2020-05-30
字数统计: 173 | 阅读时长 ≈ 1

内存溢出 out of memory

是指程序在申请内存时，没有足够的内存空间供其使用，出现out of memory；比如申请 了一个integer,但给它存了long才能存下的数，那就是内存溢出。

内存泄露 memory leak

是指程序在申请内存后，无法释放已申请的内存空间，一次内存泄露危害可以忽略，但内存泄露堆积后果很严重，无论多少内存,迟早会被占光。
memory leak会最终会导致out of memory！ 内存溢出就是你要求分配的内存超出了系统能给你的，系统不能满足需求，于是产生溢出。
